# AutoSafeX - TestBench Generator 사양서

## 1. 개요 (Overview)
AutoSafeX는 하드웨어 디자인 엔지니어 및 검증 엔지니어를 위해 직관적인 사용자 친화적 GUI를 제공하여 **동적이고 정형화된 테스트벤치(TestBench)를 자동으로 생성해 주는 프로그램**입니다. 
사용자가 복잡한 코딩 없이 최상위 디자인 모듈(DUT)과 클럭, 리셋, 인풋 데이터 등의 주요 파라미터를 그래픽 UI 상에서 설정하기만 하면, 반복적인 테스트벤치 프레임 워크 코드를 즉각적으로 작성해 검증 품질을 향상시키고 작업 시간을 크게 단축합니다.

## 2. 주요 기능 (Core Features)

### 2.1 DUT (Design Under Test) 자동 인스턴스화
- **기능**: 사용자가 최상위 하드웨어 모듈(Verilog 기반 등) 파일 혹은 모듈명을 입력 시 구문을 자동 파싱(Parsing)합니다.
- **동작**:
  - 모듈의 Port(Input, Output, Inout) 및 Parameter를 자동으로 추출하고 인식합니다.
  - 테스트벤치 템플릿 내에 해당 모듈을 정형화된 형태로 자동 인스턴스화(Instantiation)하는 코드를 삽입합니다.
  - 각 포트 연결에 필요한 Wire / Logic / Reg 변수를 자동으로 선언하여 DUT 포트와 매핑합니다.

### 2.2 Stimulus (입력 자극) 기능 자동 및 랜덤 생성
- **기능**: 인식된 입력 포트들에 대해 사용자가 원하는 데이터 입력 패턴이나 무작위(Random) 환경을 구축합니다.
- **동작**:
  - 데이터 버스 및 제어 신호에 대해 무작위 랜덤 값 입력 로직(`$urandom`, `$random` 등)을 자동 적용합니다.
  - 초기화된 테스트벤치 코드 내에 데이터 인가를 위한 `initial` 또는 별도 시퀀스 함수 블록을 구성합니다.
  - 핸드셰이크(Valid/Ready Protocol)나 특수 패턴의 경우, 기능 확장이 가능하도록 뼈대를 생성합니다.

### 2.3 Clock Generator (클럭 생성기) 설정
- **기능**: 테스트벤치의 기준 클럭 신호 생성을 자동화합니다.
- **사용자 입력 (GUI)**:
  - **Frequency (주파수)** 파라미터.
  - (추후 확장 가능 요소) Duty Cycle, Phase 설정 등.
- **동작**: 
  - 입력받은 Frequency를 시뮬레이션 타임 스케일에 맞는 주기(Period) 시간 단위로 자동 변환하여, 클럭 토글 모델링 코드를 `always` 구문 기반으로 생성합니다.

### 2.4 Reset Sequencer (리셋 시퀀서) 설정
- **기능**: 디자인의 안전한 초기화를 위해 리셋 신호의 타이밍을 제어합니다.
- **사용자 입력 (GUI)**:
  - 오토 리셋(Auto Reset) 활성화 토글버튼.
  - 리셋 딜레이 (Delay) 시간 설정.
- **동작**:
  - 지정된 딜레이 시간만큼 대기 후, 리셋이 릴리즈(De-assert)되는 시퀀스를 `initial` 블록에 삽입하여 시뮬레이션 진입 전 안정적인 Reset 상태를 확보합니다.

### 2.5 부가 기능 및 확장성
- **Assertion 설정**: SVA (SystemVerilog Assertion) 기능을 통한 자동 오류 감지 블록 템플릿 지원 여부 확장.
- **실시간 미리보기 (Waveform Preview)**: GUI 내에서 사용자가 설정한 Clock, Reset, 데이터 신호 등의 대략적인 파형 모습을 시각적으로 즉시 확인 가능.
- **템플릿 저장 (Template Save)**: 현재 설정된 파라미터 조합을 템플릿으로 저장해 두어, 이후 유사 프로젝트나 모듈에 바로 불러와 재사용 가능.

## 3. 프로그램 아키텍처 및 템플릿 구조

### 3.1 출력용 테스트벤치 템플릿 구성도
생성되는 테스트벤치 코드는 일관된 가독성을 위해 아래와 같은 표준 구조를 가집니다:
1. **Timescale & Includes**: 시뮬레이션 시간 단위 및 패키지 정의
2. **Signal Declarations**: DUT와의 연결을 위한 클럭, 리셋, 인풋/아웃풋 더미 신호 선언부
3. **DUT Instantiation**: 모듈 인스턴스 및 포트 바인딩 매핑부
4. **Clock Generation Block**: 주파수 기반 자동 클럭 토글 구문
5. **Reset Generation Block**: 지정된 Delay 사이클에 따른 리셋 제어 구문
6. **Main Stimulus Block**: 시스템 초기화 및 랜덤 입력 시퀀스 뼈대 구문

### 3.2 개발 프레임워크 (권장사항)
- **GUI**: PyQt5, PySide6 등이나 웹 기반 크로스 플랫폼 스택 적용 (사용자 친화적이고 미려한 UI/파형 프리뷰 제공 목적).
- **파서(Parser) 백엔드**: Python 정규표현식 혹은 파서 라이브러리(PyVerilog 등)를 구동하여 하드웨어 언어 텍스트에서 모듈 정보를 추출.
